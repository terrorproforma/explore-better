# Explore Better Product Testing

This is the working ledger for systematic product evaluation. Passing a code-level smoke test is supporting evidence, not proof that a feature is understandable, useful, responsive, or reliable in normal use.

## Mission

Exercise every user-facing feature and evaluate:

- purpose and discoverability;
- correctness and useful feedback;
- interaction latency and sustained performance;
- keyboard and accessibility behavior;
- failure, recovery, and cancellation paths;
- MCP visibility and safe AI operation;
- regression coverage after fixes.

## Test Method

For each feature:

1. State the user problem and expected outcome in plain language.
2. Exercise the primary mouse, keyboard, command-palette, and MCP paths that apply.
3. Record the visible loading, success, empty, disabled, cancellation, and error states.
4. Check filesystem, saved state, operation journal, or external handoff results where applicable.
5. Capture timings for startup, navigation, search, rendering, long-running work, and cancellation.
6. Decide whether the feature belongs in the product in its current form.
7. Fix prioritized issues, add a targeted regression check, and retest in the local Electron build.

## MCP Product Standard

MCP should be the fastest safe way to understand the running product. At minimum, live context should make these states inspectable without screenshots:

- active pane, tabs, paths, focus, and selection;
- open modal or panel, its title, summary, loading/error state, and bounded control list;
- current app status, visible toast, and active operation;
- last meaningful user interaction and resulting context revision;
- semantic action availability, disabled reasons, correlation IDs, visible outcomes, and stale-context rejection;
- event-driven waits and URI-only resource notifications for context and authorized operations;
- Navigator visibility, section order/counts, and scroll ownership geometry;
- path-free Health component and scheduler status;
- terminal visibility and high-level session state, without exposing terminal contents or control;
- strict redaction of paths and content outside the profile's effective roots.

MCP gaps are product findings, not testing inconveniences.
